## CRITICAL Issues
*Must be resolved before proceeding*

### Completeness Audit Findings

**C-1: Incorrect codebase analysis - `internal/web` claims are incomplete**
**Source:** spec-auditor

The spec lists files to migrate in Stage 3 but misses that `NewPluginRegistry` in `internal/web/plugin.go` also takes a logger parameter and needs updating. The spec correctly identifies these files but the migration list is incomplete.

**Impact:** Implementation will fail to compile if `PluginRegistry` initialization isn't updated.

**Fix:** Add `internal/web/plugin.go` to migration list with `NewPluginRegistry` signature change.

---

**C-2: Missing entrypoint - `recall.go` has no logger initialization**
**Source:** spec-auditor

The spec lists `internal/cli/recall.go` at Stage 2 with comment "Add initialization if missing". However, `recall.go` has NO logging code - no `SetupLogger` call, no logger variable, no `slog` import.

**Impact:** The spec implies there's a logger to migrate. There isn't. This is either scope creep (adding logging where none exists) or an error.

**Fix:** Either remove `recall.go` from migration list (if out of scope) or explicitly state "Add logging to recall commands" as a separate enhancement.

---

### AI Effectiveness Audit Findings

**C-3: Incorrect File List for Stage 3 Migration**
**Source:** ai-synergy-auditor

The spec lists config files for migration but the **callers are in `internal/cli/serve.go` and `gui.go`**, not in the config package itself. The config functions use the passed logger but don't store it.

**Impact:** Agent would make correct changes but might miss that CLI files need updated calls.

---

**C-4: Missing Entrypoints in Stage 2**
**Source:** ai-synergy-auditor

The spec lists three entrypoints but `recall.go` doesn't call `logging.SetupLogger()` anywhere. Additionally missing:
- `internal/cli/db.go` - uses storage config without logging
- `internal/cli/memory.go` - same pattern

**Impact:** Agent would attempt to add `InitLogger` to recall.go without understanding the current strategy.

## MAJOR Issues
*Should be resolved before approval*

### Completeness Audit Findings

**M-1: No FR-to-test traceability**
**Source:** spec-auditor

The spec template requires formal Functional Requirements (FR-001, FR-002, etc.) with test mapping. The spec has "Behavior Preservation Criteria" but no formal requirements.

**Fix:** Add formal requirements section with testable FRs.

---

**M-2: Thread safety claim is incomplete**
**Source:** spec-auditor

The spec states no mutex is needed, which is technically incorrect for Go's memory model without explicit synchronization guarantees.

**Fix:** Add explicit constraint: "InitLogger() MUST be called before any goroutines that might call Logger()."

---

**M-3: Helper function migration is incomplete**
**Source:** spec-auditor

The spec shows `LogToolCall` migration but `LogDBOperation` isn't mentioned.

**Fix:** Add `LogDBOperation` to the helper function migration list.

---

**M-4: Missing test isolation strategy for parallel tests**
**Source:** spec-auditor

No specification for HOW non-parallel test execution will be enforced.

**Fix:** Specify concrete approach (comments, `-p 1` flag, or build tags).
