## CRITICAL Issues
*Must be resolved before proceeding*

### AI-Readiness Findings

**CRIT-001: No Codebase Location References**
- Spec references importer, database, CLI without file paths
- Agent has no idea where to start implementation
- **Fix**: Add Codebase Context section with paths

**CRIT-002: Undefined "Last Entry ID" Concept**
- Spec uses "entry ID", "sync point" without defining JSONL schema
- What field is the ID? `uuid`? `id`? `conversation_id`?
- **Fix**: Add JSONL entry schema definition

**CRIT-003: "Backwards Scan" Algorithm Not Specified**
- "Scan backwards" is algorithmically ambiguous
- Reverse byte order? Iterate from last line? What about large files?
- **Fix**: Provide algorithmic guidance or leave to implementation plan

## MAJOR Issues
*Should be resolved before approval*

### Completeness Findings

**MAJ-C001: Divergence Marker Behavior Undefined**
- Where stored? How surfaced to users? Lifecycle?
- **Fix**: Specify storage, display mechanism, resolution

**MAJ-C002: Concurrent Import Handling Contradicts Itself**
- Edge Cases says "file-level locking or advisory lock"
- Out of Scope says "advisory locking only"
- **Fix**: Clarify stance on concurrent imports

**MAJ-C003: Missing Non-Functional Requirements**
- No memory limits, interrupt handling, atomicity
- **Fix**: Add NFR section

### AI-Readiness Findings

**MAJ-AI001: Database Schema Changes Not Specified**
- Two new entities described but no DDL, migration strategy
- Column types undefined (`file_mtime` type?)
- **Fix**: Provide explicit SQL schema

**MAJ-AI002: Output Format Undefined**
- "Reduced verbosity" doesn't specify actual format
- Progress bar? Summary? What info per import?
- **Fix**: Provide example output

**MAJ-AI003: Multiple Files vs Single File Confusion**
- Sometimes "history.jsonl" singular, sometimes "files" plural
- File discovery mechanism unclear
- **Fix**: Clarify file structure
